《算法导论》第二章：数学归纳法

Posted on 2020-01-18 | In 阅读书籍《算法导论》 | | Visitors:

第一章的简要介绍

第一章的内容比较简洁，这里简单提一下。首先在第一章中，作者简单概括了算法设计的基本步骤，分别是：设计，正确性证明，分析和实现。而本书主要集中于讨论第一个阶段，即设计。另外作者在第一章中也明确说明了本书主要讲明数学归纳法的思想在算法设计中的作用。并提前告知了虽然并非所有算法设计都可以基于数学归纳法进行解决，但是绝大数算法可以从数学归纳法作为切入点进行思考。那么接下来第二章的内容就是讲述常见的数学归纳法了。

(OK,这里我们需要留着一个问题去读书籍：既然并非所有算法都适合数学归纳法，那么什么时候适合用数学归纳法分析，什么时候不合适。是否有好的判别方法呢？)

第二章数学归纳法:

数学归纳法基本原理:

设想要证明的定理为T，若T中含有变量n，n取一切自然数（即正整数），那么证明对于所有的n，T都成立，只需要证明如下：

  • 1.当n=1时，T成立
  • 2.对于任意的i>1,如果n=i-1时T成立，可以推出n=i时，T也成立

这里我们n=1的情况往往是比较好证明的，然后我们假设n-1时，T成立的条件，去推导出n时T也成立。这样就可以从n=1，推出n=2 T成立，从n=2，推出n=3成立，以此类推从而证明对于所有的n，T都成立。

这里有个注意点：n为自然数，思考如果欲证明的自变量n不是自然数的定理T，是否可以使用数学归纳法呢？

这个基本定义比较简单，就不举例说明了。

数学归纳法变种一：

对于基本原理，用n和n+1 代替 n-1和n，那么可以很自然的有如下的变种：

  • 1.当n=1时，T成立。
  • 2.对于任意的i>1,假设n=i时T成立，可以推出n=i+1时 T 也成立。那么T对于任意自然数都成立

数学归纳法变种二（强归纳假设）：

  • 1.n=1时,T成立

  • 2.如果对每个i(i>1), 假设自然数m是任意的小于i的值，当n=m时T都成立。能够推出n=i时，T也成立。那么可以证明T对于任意自然数都成立。

    可以看出，由于这里的假设条件变强了，是n等于小于i的值都假设P能成立。而基本的归纳假设紧紧是n=i-1时，假设T成立，是强归纳假设的其中一个子假设。故名为强归纳法。而这个变种成立也是显而易见的，n=2时，由于n=1成立，所以n=2成立。n=3时，由于n=1，2成立，所以n=3成立。以此类推。

数学归纳法变种三（分块归纳法）:

  • 1.n=1时，T成立。 n=2时，T成立。
  • 2.如果对每一个n(n>2),若n-2时，T成立能够推出n时 T也成立; 那么对于任意自然数，T都成立。

这个归纳法可以将自然数分为奇数偶数的情况来证明，当n=1时成立，n-2假设成立，可以推出n=3成立，即奇数都成立；同样n=2时成立，n-2假设成立，推出n=4成立，偶数都成立。所以所有自然数都成立。

注意：既然是分块归纳，那么分块的方式会根据具体所证明内容而发生变化。相对应的证明要求也不一定是所有自然数，有可能是分块后的整数。例如上面的单独证明奇数或者偶数。如下面是常见的一种数学归纳法的证明：

  • 1.n=1时，T成立
  • 2.n(n>1 且 为2的整数幂)，假设 n/2时命题T成立，可以推出n命题P也成立，那么对任意一个2的整数幂的自然数，T都成立。

应用：

归纳技巧1：通过特例得到假设，并用归纳法证明：

例子如下：假设要计算和T(n) = 8+13+18+23+…+(3+5n), 如何有效进行使用数学归纳法：

  • 1.通过特例找到假设：根据1+2+3+4…+n的数列的和为$n^2/2+n/2$ 与题目的序列是同一个幂级数级别，不防假设T(n) 也满足二次多项式表示 $G(n) = c1 \times n^2+c2 \times n + c3$ 通过n=0，n=1，n=2的特例，带入G(n) 解决方程组，获得c1，c2，c3
    分别我2.5，5.5,0.所以得到假设$ T(n) = 2.5n^2+5.5n$
  • 2.通过数学归纳法证明此假设。

归纳技巧2：多多运用等价证明

例子：任意两条不平行，任意三条不想交的n条直线，试用归纳法证明，其会将平面划分成n(n+1)/2+1。直接使用归纳假设时比较困难的额，我们可以通过特例推导将问题归约成：证明：添加第n条直线，可以增加n个区域。
所以证明变为：

  • 假设第n条直线，使得平面增加了n个区域，去证明添加第n+1条直线，使得平面增加了n+1个区域

归纳技巧3：n表示某对象的数量，那可以假设第n个对象不存在，去探索n+1和n的关系

如技巧2中的例子，如何从第n条直线使得平面增加了n个区域，推导出第n+1条直线，平面增加n+1个区域呢？假设没有第n条直线，直接放入第n+1条直线，根据假设，应该也只是添加了n个区域，所以第n条直线的加入，使得第n+1条直线多分出了一个区域。故证明可以变为：

  • 第n条直线的存在使得第n+1条直线多分割了一个区域。

数学归纳法常见错误：

1.忽略的假设条件：

举例来说：要证明的定理的图只被假设成连通的，在证明过程中却忘了在图的变减少后检验图的连通性。这导致了归纳假设的错误。所以准确定义归纳假设时十分重要的。

2.从假设n到n+1中，n+1的情况必须是任意的。

即在n+1的时候，你不能使用特殊情况的n+1来进行推理。比如在图形归纳假设的问题中，n+1条直线的位置不能是有特殊性质的。而你推论的时候把它放在了特殊位置来进行推理

3.证明过程，忽略了的基本n=1，n=2情况的验证。

有些命题在推论过程中需要的是n要大于3，甚至n要大于比较大的数才成立。而你只是验证了n=1的情况显然成立。那么就会导致错误的推论。

《算法导论》第三章：算法分析

Posted on 2020-01-18 | In 阅读书籍《算法导论》 | | Visitors:

本章内容概括

算法分析主要的目的就是分析其算法的复杂度。如何脱离硬件去度量不同算法的效率呢？本书主要讲述最坏情况的估计方法。而最坏的情况往往是输入规模n非常大的情况。接下来，我主要描述这种分析方法，然后归纳出常见算法的复杂度的数学公式。

大O符号和Ω符号:

大O符号定义：

对算法进行运行时间评估的时候，由于都是考虑n非常大的情况，常数项往往被忽略。为了对此进行有效的说明，所以需要引入一些特别的符号。首先，如果存在常数项c和N，使得对于所有的n>=N,有g(n)<=cf(n),则称函数g(n)相对于另一个函数f(n)是O(f(n)).从这个公式可以看出，对于足够大的n，函数g(n)不超过函数f(n)的一个固定倍数。函数g(n)可能比cf(n)小。符号O是从上限进行约束。

大O符号性质与常见推论与定理：

  • 1.O中包含常数项没有意义，所以我们有O(n) 而不写诸如O(5n+4)
  • 2.对数不必写出底数，因为不同的底数，根据换底公式也只是常数项的区别，故写为O(logn)
  • 3.O(1) 用来表示常数项
  • 4.指数函数要比一个多项式函数增长得快，即$ (f(n))^c = O(a^{f(n)})$ 如果用n代替f(n)，对于所有常数c>0和a>1,有$n^c=O(a^n)$
  • 5.线性函数比一个对数函数增长得快，推导：如果用$log_a^n$ 代替推论4公式中的f(n),对于所有常数c>0和a>1,有 $(log_a^n)^c=O(a^{log_a^n})=O(n))$

  • 6.大O符号基本性质：满足加法律和结合律，由于O符号表示<=,注意不能定义减法和除法：

    1.如果 f(n)=O(s(n)) 并且 g(n)=O(r(n)),则f(n)+g(n) = O(s(n)+r(n))
2.如果 f(n)=O(s(n)) 并且 g(n)=O(r(n)),则f(n)*g(n) = O(s(n)*r(n))
  • 7.根据定义，大O符号可能会存在很上限太大，没有实际意义的问题。

其他符号

  • 1.下界Ω符号：如果存在常数项c和N，使得对于所有的n>=N,在输入规模为n时解决问题的步数T(n)至少为cg(n),则我们说T(n)=Ω(g(n)).
  • 2.等效θ符号，如果f(n)同时满足:f(n)=O(g(n)),f(n)=Ω(g(n)),则可又记为f(n)=θ(g(n))

由于我在公司的职位现阶段的职位是模型结构搜索方向的算法工程师，业内的人士都清楚这个方向是比较耗费硬件资源，特别是GPU。比如一个算法可能就会需要同时32张GPU卡跑一周才出来一个实验结果。今天恰好在公司听到几位同事在讨论公司最近正在修改硬件资源分配的策略的问题。现在的情况是公司整体的硬件资源不够。公司的策略是按照优先级进行排序，对于某一个人的是第一个任务那么提交的时候就放在第一梯队，这个人的第二任务提交就放在第二个梯队，以此类推。等第一个梯队的任务全部都排完之后，才轮到第二个梯队的任务进行排队。这样做很明显会出现这两个问题：

  • 1.如果某一个人的任务运行比较快，但是频率却很高，那么他的任务永远在第一梯队。导致所有人第二个任务永远在排队，别人永远都只能运行一个实验。就造成了资源分配的不公平性。
  • 2.每个人跑一个实验需要同时跑的GPU卡数是不一样，那么就会出现这种情况，就是现在公司可能可用的GPU卡数只剩下了24张，此时第一梯队第一个任务的实验可能需要32张卡，那么现有公司策略是等待，等待什么时候GPU卡数满足了32张卡，执行完第一梯队的第一个任务后，在轮到第二个任务的执行。那么在等待的过程中，24卡的资源就白白浪费了。

那么有什么方法可以解决这种问题呢？实际上要从根本解决就是扩充公司硬件资源。但是在硬件资源一定的情况下，又要如何缓解这种问题？

我这里有一个不太成熟的想法是这样的，简单来说：首先需要统计下每个人在N天得到一个实验结果需要多少卡。如果资源少，那么N在满足进度的前提下可以适当的大点，如果资源多，项目时间紧，那么N就适当的小一点。然后就设置某些卡的优先级让这个人在这些相应的卡的任务的优先级相较其他人都最高。如果这些卡没有人用，其他人需要跑任务的话，那么在按照最开始设定的排队策略来跑其他人的程序。

上面的想法可以概括成，不同的GPU卡的优先级设置是不同的，组织规模的从大到小排序分别是：公司，院，部门，组，个人。那么个人先在自己的优先级卡先跑，如果不够则会占用同组的资源卡池，但是优先级会降一等，同组的不够就会尝试去占用同一部门的资源卡池，但是此时任务的优先级会在降一等。以此类推。通过这种方法来缓解资源占用不均匀的情况。比如上述的第一点，某一个人的任务运行比较快，频率高，那么他就会在自己的优先级卡时里面跑，想跑其他人的卡的话优先级就降了，而减少了对其他人的影响。又比如上述的第二点也是如此。

当然这样做必然引入了其他的弊端。比如增加了维护和开发的成本等等。

最后，我相信万物的发展规律一般都存在J字曲线，即 刚开始采取某个正确改变的时候，由于需要适应期，可能产出还不如之前已经习惯的策略。但是一旦熬过了这个阶段，那么改变的优势就会很好的体现出来。我们公司的目前就处在习惯的阶段，相信以后会越来越好。

另外这种分配策略，用在其他资源上是不是也一样呢？在其他资源上是否存在共赢的局面，而非这种零和博弈的情况呢？以后有机会在好好思考一下。

公司年会

Posted on 2020-01-16 | In 每日感悟杂谈 | | Visitors:

公司年会：

今天是公司举办年会的日子，我们公司有个传统，就是公司举办活动每次由新一届的小管来全程组织，主要由新入职的同事进行节目表演。由于下一年我也是小管，需要负责安排，那么就简单的讲解一下今天对于公司这次举办年会的感受，首先总整体上来看还是组织的还是很用心的，比如撒金沙来进行点亮公司的标语很有代表性。用抽签的方式强制进行游戏环节也是比较让人接受的。有几个小组的表演非常用心。其中印象最深刻的就是描述公司文化场景的沙画，另一个印象深刻的就是自己的原创歌曲。这说明了一个体现了公司文化的表演形式和原创的敬畏之心，对节目效果的增益。那么接下来需要说明一下需要改进的地方：

  • 1.年会的时候公司的位置摆放实际上并没有之前的效果好，今天的位置摆放是想学术会议厅那样，并不适合后排的同事观看场地的表演。特别是其中有个节目是进行非普通话配音的，在不看字幕的情况下完全都不知道他在说什么，导致节目效果很差。去年年会那种垂直中间有个桌子，面向表演台则有更好的观感体验，或者需要的情况下，字幕放在高处。
  • 2.领导在讲话的时候，居然没有准备润喉的饮品
  • 3.中间的游戏环节全部都是竞手速的，实际上游戏效果并不好
  • 4.也是这次比较失败的一点，现场节目大部分效果不理想的原因是彩排时间太少，很多节目的演出特别是一些灯光，一些有乐器的声音设备，实际上有很多问题都需要在现场进行调试。但是如果资金的限制，如何在半天之内就进行彩排好是一个严峻在来年也需要重点解决的问题。
  • 5.领导和下属的互动比较少。

好了，今天总结主要从我直观的感受的角度切入，略微肤浅。等以后有机会就更加深入的思考一下，比如从目的性角度切入，为什么要举办年会，今天举办的年会到底有没有真正的实现这些目的。如果没有实现是为什么呢，如何改进？

启动会

Posted on 2020-01-14 | In 每日感悟杂谈 | | Visitors:

2020-1-14：

  • 早上: 早上参加了公司氛围管理组织的启动会。启动会的内容主要是总结去年的工作，对新一年工作的期望，对新加入的同事，告知本组织的架构，和主要负责内容。也相当于是第一次正式将组织的人员全部召集在一起。但是整体会议没有讨论具体的负责事宜，具体每个组织内部各个部门会再开展会议进行详细的宣贯和探讨。实际上在场的很多人都在疑惑这个会到底有没有必要的开？ 这里主要有两种途径来思考，首先是否必要，可以从这个活动是否可以被另一个形式来代替，以达到相同的效果，比如直接发邮件。或者先找到这种会议的作用，而作用依赖于这种会议形式。其中如何找到作用呢？ 你可以找到类型的事务，比如和这个会议很像的开学典礼，那么讲述开学典礼的作用。或许就能迁移至这种会议的作用了。那么开学典礼有哪些作用呢？1.归属情感凝聚功能，2.文化认同功能.3.激发动力功能.等等

  • 下午 今天是和另一个同事最后一次举办健康活动，之后就会移交给新入职的同事。总结这一年的健康活动，它的动机其实很简单：就是希望同事们在每周久坐之后，提醒他们起来走动走动，做一些小运动，听听歌，吃些水果。在这一年的活动中，我获得了如下的收获：1.领导的带头作用效果是比较明显的，以后举办类似活动，最好需要先和领导沟通，让他们先配合。2.需要尽量在多的场合，比如邮件，微信群等，宣传很多人参加的场景，让大家觉得其他人都在参加。3.奖励和小活动需要多样化。4.音乐实际上挺重要的，当活动开始的时候，你如果用喊的叫大家来活动，或许除了自己熟悉的同事可能会参加，其他同事可能来参加的很少，而且无疑也增加了你的工作量。而放音乐则一方面可以有效调动大家运动的心情，另一方面自然而然的作为活动开始的信号。5.但是音乐选择也是有讲究的，尽量不要选择高音的，或者很嘈杂的音乐，多选择平缓或者欢快的音乐。6.音乐也存在审美疲劳，可以每4周为一循环，每周根据时间如果10分钟就准备3首，这样共12首的音乐，每次活动循环3首播放。隔较长时间在进行更改12首歌。虽然已经举办了一年，但是我还是会思考，这个活动是否还有存在的意义，是否还需要改进。我也一直在询问自己这四个问题：每周就一次这样的活动是否可以让同事更健康？ 造成久坐，或者职场不健康的根本原因是什么呢？我们的活动多大程度上缓解了这个问题呢？举办我们这个活动是否有什么反而损失掉了呢？

  • 晚上 从数据采集的角度，总算找到了昨天性能不佳的问题所在，主要的原因就是测试采集的数据和训练采集的数据不一致的问题。定位完问题，就开始进行相应问题的解决了。所以遇到问题，入手的第一步，就是定位问题。最后，晚上和师姐一起回家，路上讨论了论文写法的重要性，你平时会怎么看一篇论文，一般先看简介，看贡献等再看实验性能。那么你自己在写论文的时候，也要在这相应的地方多花时间，仔细琢磨。所谓从评审的角度来审视自己的论文，思考问题如果让自己来审，会提那些问题，会有哪些实验希望补充。

从组会看自己

Posted on 2020-01-13 | In 每日感悟杂谈 | | Visitors:

2020-1-13：

— 今天早上：开了组会，报告了上一周的内容，其中深刻体会到了一点，就是自己所报导的事情，领导并不是很关注，反而自己觉得次要的地方，领导进行细节询问。结果由于自己不重视，细节的提问也没有回答上来。所以从这里我复盘了一下，得出了两个可以改进的地方：首先是：领导并没有意识到我工作的重要性，或者说目的，可能需要在报告中详细提出，让领导觉得上周内容有进步并且有价值。另一个问题是：我并不知道领导最近在思考，公司最近最需要解决的问题是什么？所以这样使我思考的方向，判断工作内容的重要程度的评价方法和领导有所偏差，以后需要和领导多沟通，并了解公司具体需要哪些问题进行解决。

—今天下午：进行了专利的修改，发现写专利的话，并不是有一个点创新，写清楚就可以。你需要写的是你解决问题的一整套方案。所以你必须写清楚每个步骤，即使有些步骤不是你要保护的权利要求。另外，专利的上位虽然是交给专利代理人的，但是你也需要自己进行相关的上位，专利代理人写完初稿会叫你进行修改，主要查看自己保护的点是否都在专利保护范围，是否有一些细节没有写清楚，或者写的太清楚。是否存在现有方法无效掉专利的情况等。今天自己修改完专利后，找了主管进行审阅，发现自己看的还是不够仔细，重点看权利要求，需要进行每个句子和词的仔细揣摩，判断当前词是否有歧义等。而且发现有些词的理解你和代理人是存在区别的，你觉得可能存在歧义，但是代理人觉得不会有，这个时候，我的做法是听代理人的。但是也不知道这样做是好还是坏。

—今天晚上：最近正在忙着做论文的实验，但是实验结果却不是很理想，之前认为在随机采集的数据集上进行测量的结果，性能很好。但是今天发现几个数据上体现的误差极大。这说明之前的实验的论证实际上是存在不妥的地方的。测试集无法很好的验证模型在各个样本上的性能。那么一样，我们需要先进行问题的定位，实验结果查无非是如下几个条件不满足造成的实验结果:

  • 算法本身问题：是否是因为算法有效的前提条件本身不满足
  • 数据输入的问题：是否是因为数据集采集不全面，和训练数据分布差异过大
  • 训练的问题：训练是否有收敛，或者过拟合了

  • 测试的问题：是否训练的数据和测试的数据之间保持了一致性，比如预处理等

    明天从这四个方向好好确定下问题所在，在根据问题进一步的改进。

—今天深夜：听了大咖的一些阅读分享，收货很大，简要的进行两点进行总结：

  • 1.共有信息到大众信息转化的重要性，即有些话需要直接表达出来，不要去猜对方，例如领导，朋友的想法。
  • 2.未来的核心竞争力，可能并不是你去掌握多少种技能或者技术，而是如何调度周边的技能或者技术，进行整合资源，创造出更高价值的能力。
